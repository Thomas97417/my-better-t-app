# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Commands

```bash
bun install              # Install dependencies
bun run dev              # Start all apps (web + Convex backend) via Turborepo
bun run dev:web          # Start only the web frontend (port 3001)
bun run dev:server       # Start only the Convex backend
bun run dev:setup        # Initialize/configure Convex project (first-time setup)
bun run build            # Build all applications
bun run check-types      # TypeScript type checking across all workspaces
```

Convex environment variables from `packages/backend/.env.local` must be copied to `apps/web/.env`.

## Architecture

**Monorepo** using Bun workspaces + Turborepo. Generated from Better-T-Stack.

### apps/web — React + TanStack Start frontend
- **React 19** with **TanStack Start** (SSR meta-framework), **TanStack Router** (file-based routing), **TanStack Query** (data fetching via `@convex-dev/react-query`)
- Routes live in `apps/web/src/routes/`. The route tree is auto-generated in `routeTree.gen.ts` — do not edit manually.
- UI: **Tailwind CSS v4** + **shadcn/ui** components in `apps/web/src/components/ui/`. Config in `apps/web/components.json`. Path alias `@/*` maps to `src/*`.
- Auth client: `apps/web/src/lib/auth-client.ts` (Better-Auth browser client with Convex plugin). Server-side auth: `apps/web/src/lib/auth-server.ts`.
- Root layout (`__root.tsx`) fetches auth token in its loader and provides Convex auth context. Protected content uses `<Authenticated>` / `<Unauthenticated>` components from `convex/react`.

### packages/backend — Convex backend
- Convex functions (queries, mutations, HTTP routes) in `packages/backend/convex/`.
- `_generated/` directory is auto-generated by Convex — do not edit.
- Auth: **Better-Auth** with `@convex-dev/better-auth` adapter. Config in `auth.config.ts`, client in `auth.ts`, HTTP routes in `http.ts`.
- `schema.ts` defines the Convex database schema (currently empty — add app tables here).
- Convex queries use `ctx.db` for database access. Backend functions are imported on the frontend via the generated `api` object.

### packages/env — Environment validation
- Uses `@t3-oss/env-core` + Zod to validate `VITE_CONVEX_URL` and `VITE_CONVEX_SITE_URL`.

### packages/config — Shared TypeScript config
- `tsconfig.base.json` extended by all packages. Strict mode, ESNext, bundler resolution.
